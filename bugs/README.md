# Bug Tracking

This folder contains documentation for bugs encountered during development.

## Bug Report Format

Each bug report follows this naming convention:
- `XXX-brief-description.md` where XXX is a zero-padded 3-digit number

## Bug Report Template

When creating a new bug report, include:

1. **Status** - OPEN, IN_PROGRESS, RESOLVED, WONTFIX
2. **Date** - When the bug was discovered
3. **Severity** - LOW, MEDIUM, HIGH, CRITICAL
4. **Description** - What the bug is
5. **Error Details** - Full error messages and stack traces
6. **Root Cause** - Why the bug occurred
7. **Environment** - Relevant system/software versions
8. **Steps to Reproduce** - How to trigger the bug
9. **Solution** - How to fix it
10. **Prevention** - How to avoid it in the future
11. **Related Files** - Code/config files involved
